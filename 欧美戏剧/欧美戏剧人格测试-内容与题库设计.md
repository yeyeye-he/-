## 二、人格模型设定（四维 × 二极）

> ⚠️ 下列维度为**戏剧观看与讨论习惯**上的趣味分类，不对应临床或职业 psychology。

每一维在计分上独立累加，最后比较该维两端得分高低，决定取哪一极（平局规则见后文）。

### 维 1：在场方式 — **I（Immersion 沉浸）** ↔ **B（Brechtian 间离）**

| 极 | 含义（剧场外行话） |
|----|---------------------|
| **I** | 容易被情绪带走，愿意「相信此刻」，散场后情绪仍在。 |
| **B** | 本能追问形式、立场与历史语境，「我在看戏，也在看这台戏怎么被做出来」。 |

### 维 2：剧目光谱 — **C（Canon 典册）** ↔ **E（Edge 前沿）**

| 极 | 含义 |
|----|------|
| **C** | 古希腊/莎士比亚/现实主义经典、名团驻场、愿意啃长剧与厚重文本。 |
| **E** | 当代文本、跨媒介、形体/后戏剧/实验空间，越「不像传统戏」越兴奋。 |

### 维 3：剧场社交 — **S（Social 聚场）** ↔ **Q（Quiet 静退）**

| 极 | 含义 |
|----|------|
| **S** | 中场必聊、散场聚餐、爱写流水账剧评、票根社交。 |
| **Q** | 更喜欢独自看或极少交流，散场需要独处消化。 |

### 维 4：价值裁判 — **T（Text 文本派）** ↔ **G（Gesamtkunstwerk 舞台整体派）**

| 极 | 含义 |
|----|------|
| **T** | 「台词/结构是否忠于意图」权重大；改编动大刀会纠结。 |
| **G** | 声光舞美节奏统合更动人即可；文本是材料之一，不是唯一裁判。 |

### 十六型编码规则

按顺序拼接四维字母，例如 **ICST** = Immersion + Canon + Social + Text。

> 正式对外可用**中文昵称 + 英文代号**，英文代号便于生成「结果图」与话题传播。

### 4.1 逐条解释文案（`SUB_DESCRIPTIONS`）

结果页折叠区标题为 **「十五个观剧心理侧写」**：在条柱下方展示与上表 1–15 一一对应的**短说明**，用语刻意勾连 **剧场常识**（共谋、间离、现场声画、改编权、小剧场经济等）与 **日常心理语言**（情绪感染、注意分配、延迟满足、经验开放性、认知闭合、内向恢复等），并反复强调 **仅供娱乐自测，不构成心理诊断或人格定论**。

- **定稿位置**：`dramati/questions_bank.py` 中字典 `SUB_DESCRIPTIONS`（键 `1`…`15`）。  
- **API**：`GET /api/meta` 返回 `sub_descriptions`（字符串键 `"1"`…`"15"`）；`POST /api/score` 在 `result.sub_dimensions[]` 每条中带 `description` 字段，与前端条柱同步渲染。  
- **维护约定**：若改子维含义或增删子维，须同步改 `SUB_LABELS`、`SUB_DESCRIPTIONS`、`compute_scores` / `sub_maxima` 与本文第四节。

## 五、题库总设定

### 5.1 题目数量、内部题卷与三种观戏人群

| 内部键 `group` | 组成 | 题量 |
|----------------|------|------|
| `main` | Q01–Q32（习惯向主干） | 32 |
| `extra` | Q33–Q40（轮换/生活小品） | 8 |
| `content` | Q41–Q52（戏剧内容加餐） | 12 |
| `main_extra` | main + extra | 40 |
| `main_content` | main + content | 44 |
| `all` | main + extra + content | 52 |
| `shakespeare` | 莎士比亚专题专卷（`_questions_shakespeare`，与上表无包含关系） | 18 |

**前台三种人群**（`AUDIENCE_PRESETS`，参数 `audience`）：用户只选一种身份，后端映射到上表题卷。

| `audience` | 对外文案（摘要） | 映射 `group` |
|------------|------------------|--------------|
| `fan_rare` | 心头好·稀客型：喜欢戏但很少进场，按「假如今晚我坐在剧场里」作答 | `main`（32） |
| `curious` | 名段略懂型：课表/节选级，生活轮换题有，不含 12 道文本专梗 | `main_extra`（40） |
| `regular` | 现场常驻型：老观众，习惯+生活+课纲向文本题最全 | `all`（52） |

调试或扩展仍可直接调用 `GET /api/questions?group=…`；正式产品路径为 `?audience=fan_rare|curious|regular`。莎士比亚专题页走 `?group=shakespeare`（见第十二节）。

### 5.1a 题干口味（与戏剧现场对齐）

主干 `main` 卷题干优先锚在 **剧场暗场、当季排期、海报宣发、散场余波、固定观剧搭子、戏剧课/剧社、话剧票** 等语境，减少纯生活泛化句；选项与赋分仍以 `questions_bank.py` 为准。文档第六节中的「题库草案」在字面上与代码可能有个别措辞迭代，**以代码为准**。

### 5.1b 流程题 / 彩蛋（可选，未实现）

- 流程题：如「是否首次看现场默剧」仅改结果页导语，**未写入当前代码**。
- 隐藏分支：如剧组义工、剧本杀等彩蛋，**可后续加**。

### 5.2 题型与选项

- **单选题**为主：每题 **4 个选项** 或 **2×2 场景**（实现简单）。
- 每选项携带：`I/B`、`C/E`、`S/Q`、`T/G` 的分数增量（可为 0–3），以及十五维子分增量。

### 5.3 计分规则

1. 四维各端分别累加（如 `score_I`、`score_B`…最终若 `score_I >= score_B` 则第一维为 **I**，否则 **B**；其余类推）。
2. **平局**：实现为「若一端得分 **≥** 另一端则取该端」——故持平时依次为 **I**（相对 B）、**C**（相对 E）、**S**（相对 Q）、**T**（相对 G）。
3. **主类型代号**：按维 1–4 顺序拼四个字母。
4. **匹配度（如 92%）**：可用「胜出维的净差 / 理论最大差」加权平均，再映射到 70%–99%，避免全是 100%。

### 5.4 内容红线（课堂/公开展示）

- 不嘲讽具体国家、族群、残障；可嘲讽「迟到拍照开闪光灯」等行为。
- 不用真实剧团/演员作恶梗；可用「某类典型观众」泛化。

## 十一、人格「小人」形象（对齐 16Personalities 式视觉）

## 十二、莎士比亚戏剧 · 独立专题测

**定位**：在总测（三种观戏人 × 不同题量）之外，单独开一条产品线式入口，**题干全部聚焦莎士比亚**（文本想象、改编立场、经典场面与角色共情），仍落回 **同一套四维轴与十六型昵称**，便于与总测结果对照。

| 项 | 说明 |
|----|------|
| 入口 | 浏览器 `GET /shakespeare`；总首页票根下亦有链入 |
| 题量 | **18 题**（`questions_bank._questions_shakespeare`，题 id `sh_sq01`…`sh_sq18`） |
| API | `GET /api/questions?group=shakespeare`；`POST /api/score` body：`{ "group": "shakespeare", "answers": [...] }` |
| 与总测关系 | **不**混入 `AUDIENCE_PRESETS`；专卷独立，避免稀客型被莎翁专名词劝退 |

**题库维护**：新增/改写专题题时只改 `_questions_shakespeare`，并在本节更新题量数字。
